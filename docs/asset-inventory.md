# Asset Inventory

While Mr.Holmes is predominantly a CLI tool, it utilizes several internal and visual assets for its operation and its supplementary GUI layer.

## OSINT Data Assets
- `Site_lists/`: Crucial JSON files outlining logic, URL patterns, error tags, and identifiers for hundreds of target websites. (e.g., `Username/`, `Phone/`).

## Presentation & Branding Assets
- `Banners/`: Static text files acting as ASCII art banners presented at CLI startup.
- `Quotes/`: Static text files containing quotes randomly fetched to customize the starting prompt.
- `Icon/`: Iconography for the application.

## GUI Layer Assets
- `GUI/Css/`: Stylesheets defining the visual language of the PHP viewer.
- `GUI/Theme/`: Structural theme definitions or layouts.
- `Screenshot/`: Demonstration imagery showcasing the CLI and GUI in action (e.g., `Map_Test.png`, `Graph_Test.png`).

## Configuration & Localization
- `Lang/`: Text-heavy files orchestrating the robust multi-language capabilities (`Core.Support.Language`) across the CLI components.

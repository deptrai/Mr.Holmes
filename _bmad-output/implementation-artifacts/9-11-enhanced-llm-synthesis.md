# Story 9.11: Enhanced LLM Synthesis

Status: ready-for-dev

## Story

As the LLM synthesis engine,
I want to accept a `ProfileEntity` object as structured context instead of raw graph data,
so that AI reports include personality traits and behavioral insights grounded in real data.

## Acceptance Criteria

1. `LLMSynthesizer` tại `Core/engine/llm_synthesizer.py` nhận thêm optional `profile: ProfileEntity | None = None`:
   - Existing `synthesize(graph_data: dict, ...)` signature vẫn giữ nguyên — backward compat
   - Thêm `synthesize_profile(profile: ProfileEntity, raw_graph: dict | None = None) -> str` — new method
   - Epic 8 callers không bị ảnh hưởng

2. `synthesize_profile()` prompt structure:
   - Section 1 **Confirmed Data** — từ `ProfileEntity` fields (names, emails, platforms, breach_sources)
   - Section 2 **Behavioral Signals** — từ `active_hours`, `bios`, `locations`
   - Section 3 **Personality Analysis Task** — yêu cầu LLM infer Big-5 traits (Openness, Conscientiousness, Extraversion, Agreeableness, Neuroticism) dựa trên evidence
   - Section 4 **Confidence Context** — `profile.confidence` score và `profile.sources` list
   - Tất cả AI-generated content phải được label `[AI-generated hypothesis]`

3. Output format của `synthesize_profile()`:
   ```markdown
   # OSINT Intelligence Report

   ## Confirmed Facts
   - **Real Names:** Nguyen Van A (source: Maigret/GitHub, confidence: 0.85)
   - **Platforms:** Instagram, GitHub, Spotify (12 total)
   - **Breach Sources:** Adobe, LinkedIn
   - **Overall Confidence:** 0.81

   ## Behavioral Profile [AI-generated hypothesis]
   - **Active Hours:** Estimated 10pm-2am (UTC+7) based on post timestamps
   - **Location Signals:** Ho Chi Minh City (3 sources)

   ## Personality Analysis [AI-generated hypothesis]
   Based on bio patterns and activity data:
   - **Openness:** High — diverse platform presence, creative bio
   - **Conscientiousness:** Medium — irregular posting patterns
   ...

   ## Intelligence Gaps
   - Phone number: not confirmed
   - Employment: GitHub shows "FPT Software" — unverified

   *Report generated by Mr.Holmes. AI hypotheses require human verification.*
   ```

4. Prompt injection mitigation:
   - Sanitize tất cả values từ `ProfileEntity` trước khi inject vào prompt
   - Strip `</s>`, `###`, `[INST]` và các token patterns đặc biệt khỏi user-data values
   - Max length per field: 500 chars (truncate với `...`)

5. `requirements.txt`: không thêm dependency mới — dùng existing LLM client

6. Unit tests tại `tests/engine/test_llm_synthesis_epic9.py`:
   - Test `synthesize_profile()` gọi đúng prompt structure (mock LLM client)
   - Test prompt injection mitigation (input chứa `###SYSTEM` → được strip)
   - Test backward compat: `synthesize()` vẫn hoạt động với raw graph
   - Test empty ProfileEntity → report vẫn render, ghi nhận "insufficient data"

### Sanitization Helper

```python
import re

_INJECTION_PATTERNS = re.compile(
    r"(</s>|###|<\|im_start\|>|<\|im_end\|>|\[INST\]|\[/INST\]|<s>)",
    re.IGNORECASE
)

def _sanitize_for_prompt(value: str, max_len: int = 500) -> str:
    cleaned = _INJECTION_PATTERNS.sub("", value)
    if len(cleaned) > max_len:
        cleaned = cleaned[:max_len] + "..."
    return cleaned
```

### ProfileEntity → Prompt Conversion

```python
def _build_profile_context(self, profile: ProfileEntity) -> str:
    lines = ["## Confirmed Data\n"]
    if profile.real_names:
        names_str = ", ".join(
            f"{_sanitize_for_prompt(f.value)} (src: {f.source}, conf: {f.confidence:.2f})"
            for f in profile.real_names[:5]
        )
        lines.append(f"- Real Names: {names_str}")
    if profile.platforms:
        lines.append(f"- Active Platforms: {', '.join(list(profile.platforms.keys())[:10])}")
    if profile.breach_sources:
        lines.append(f"- Breach Sources: {', '.join(profile.breach_sources[:10])}")
    lines.append(f"- Overall Confidence: {profile.confidence:.2f}")
    lines.append(f"- Data Sources: {', '.join(profile.sources)}")
    return "\n".join(lines)
```

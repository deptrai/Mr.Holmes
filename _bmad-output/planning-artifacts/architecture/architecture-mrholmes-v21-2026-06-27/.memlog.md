- (constraint) Brownfield: kế thừa v2.0 architecture (MCP server + Plugin system + Evidence Store). Không pháy backward compat.
- (constraint) Plugin Protocol (IntelligencePlugin): name, requires_api_key, check(target, target_type) → PluginResult. Mọi plugin mới PHẢI implement protocol này.
- (constraint) MCP tool pattern: @mcp.tool() async def func(...) -> str. Trả JSON string. Plugin lookup qua _get_plugin(name).
- (constraint) Evidence Store: SQLite, investigation_id FK, evidence rows với tool_name/target/target_type/result_json/confidence. Audit log auto.
- (constraint) Stealth browser: Core/browser/stealth_context.py — Playwright async, start()/stop() API. Mọi scraper mới dùng StealthBrowser context manager.
- (decision) AD-1: Plugin-first architecture — mọi OSINT source mới là plugin (Core/plugins/), không phải standalone script. Plugin auto-discover qua PluginManager. Binds: tất cả sources mới tuân thủ Protocol. Prevents: divergence giữa plugins.
- (decision) AD-2: Vietnam OSINT layer — nhóm Vietnam-specific plugins (VnBusiness, VnPhone, VnCourt, VnTax, VnNews, Zalo, FacebookVn) vào stage=3 (deep enrichment). Stage router ưu tiên stage 1-2 trước, stage 3 khi có clues. Binds: execution order. Prevents: chạy Vietnam scrapers khi chưa có clue (waste).
- (decision) AD-3: Scraper pattern cho social media — dùng StealthBrowser (Playwright) cho Facebook/Instagram/TikTok/Zalo. Mỗi scraper: navigate → wait_for_selector → extract JSON-LD/meta tags/eval JS. Không dùng HTTP requests cho JS-heavy sites. Binds: scraping approach. Prevents: 403/bot detection failures.
- (decision) AD-4: Government records — scrape public portals (tracuunnt.gdt.gov.vn, thuvienphapluat.vn, dangkykinhdoanh.gov.vn) qua Playwright. Không có official API. Rate limit 1 req/3s. CAPTCHA: log warning, return partial result. Binds: gov record approach. Prevents: IP ban.
- (decision) AD-5: Wire 3 engine modules vào MCP — llm_synthesizer → synthesize_report tool, pdf_builder → export_pdf tool, mindmap_generator → generate_mindmap tool. Giữ nguyên implementation, chỉ thêm MCP wrapper. Binds: MCP surface. Prevents: dead code.
- (decision) AD-6: Cross-reference engine —新增 cross_reference tool: nhận list evidence IDs, tìm overlap (email xuất hiện trong breach + social profile), suggest new targets. Binds: investigation workflow. Prevents: manual cross-check by AI orchestrator.
- (decision) AD-7: Truecaller/GetContact integration — phone → owner name lookup qua Truecaller API (paid) hoặc GetContact (free tier). Binds: phone enrichment. Prevents: phone chỉ trả carrier mà không có tên.
- (decision) AD-8: News archive search — Vietnam news (Tuổi Trẻ, VnExpress, Thanh Niên) qua Google News API + site-specific dorks. Không scrape trực tiếp (anti-bot). Binds: news search. Prevents: IP ban từ newspaper sites.
- (decision) AD-9: Avatar reverse search — Google Images + Yandex Images + FaceCheck.id API. Input: avatar URL từ social profile. Output: other profiles dùng cùng avatar. Binds: identity correlation. Prevents: manual image search.
- (decision) AD-10: Snusbase plugin — breach DB alternative (paid, /mo). Tích hợp như plugin riêng, không thay thế IntelX/LeakLookup. User cung cấp key. Binds: breach coverage. Prevents: single point of failure.
